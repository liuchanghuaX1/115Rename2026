# 115Rename2026+

一款专为 115 网盘设计的视频整理与标准化重命名脚本。采用“本地加工”与“网络刮削”双轨分离架构，集成 **JavLibrary → JavBus → xslist** 多站信息源，智能缓存与并发控制，让批量重命名更快更稳。

> 当前版本：**1.5.0**  
> 原仓库：[https://github.com/liuchanghuaX1/115Rename2026](https://github.com/liuchanghuaX1/115Rename2026)

## ✨ 主要功能

### 🧹 本地番号加工
- **精准番号提取**：自动识别并标准化番号（如 `ABP-123`、`SSIS-456`），支持 `T28`、`S2M` 等数字混合厂牌及 `h637` 等单字母番号（自动规范为 `TokyoHot-h637`）。
- **分段处理**：自动将 `part1`、`PT.2`、`CD3` 等转换为标准的 `-1`、`-2`、`-3` 后缀。
- **日期提取**：从文件名中智能提取发行日期（如 `2024-01-01`），不依赖网络。
- **本地标题保留**：在标准化过程中保留用户原有中文标题与标记（如 `4K`、`破解`、`中文字幕`），避免信息丢失。
- **垃圾清理**：自动移除广告域名、重复番号、多余括号、视频编码信息等干扰内容。

### 🌐 多站改名（在线刮削）
- **三引擎轮询**：按 **JavLibrary → JavBus → xslist** 顺序自动获取影片元数据，一处命中即停止，保证成功率。
- **完整信息补全**：自动获取并拼接 **番号 + 正式标题 + 女优名 + 日期**，生成规范文件名。
- **智能缓存**：同一番号的数据全局缓存，后续同名文件直接本地读取，大幅减少重复请求。
- **有限并发控制**：联网改名默认并发 3，本地加工并发 5，有效避免请求过于频繁被限制。
- **可视化进度条**：实时显示当前处理进度与任务总数，批量操作一目了然。

### 🛡️ 防误伤引擎
- 采用**透视匹配算法**，精准区分相似前缀（如 `DASS` 与 `SDASS`），杜绝错误截断。
- 内置大量厂牌前缀库，覆盖主流片商与系列，持续更新。

## 📋 使用指南

1. 登录 115 网盘，进入任意目录。
2. **选中需要处理的文件/文件夹**（可多选）。
3. 在 **“更多”** 菜单中找到以下选项：
   - **本地番号加工** → 仅使用文件名中的信息进行标准化（不联网，速度快）。
   - **改名(多网站轮询)_统一格式** → 在线获取信息，补全标题、女优、日期（推荐用于命名不全的文件）。
   - **归档至文件夹** → 按女优或系列自动归类（需先设置归档根目录）。
   - **设置为归档根目录** → 右键目标文件夹，点击此项将其设为归档根。
4. 实时查看右下角的进度条，处理完成后会有成功/失败通知。

## ⚠️ 已知问题与限制
- **FC2 番号**：目前暂不支持从网站获取信息，联网改名时会自动降级为本地加工处理。
- **JavDB 评分**：暂不支持获取 JavDB 评分数据。
- **网络波动**：若三个信息源均无法访问，脚本会保留原始文件名并给出提示。

## 🚀 安装方法

1. 首先安装用户脚本管理器扩展（如 [Tampermonkey](https://www.tampermonkey.net/)）。
2. 前往 GreasyFork 页面安装：[115Rename2026 脚本主页](https://greasyfork.org/zh-CN/scripts/574424-115rename2026) （发布时会更新到最新版本）。
3. 或者直接从本仓库下载 `115Rename2026+.user.js` 文件，拖拽到 Tampermonkey 中安装。

## 📄 技术细节

- 基于 `GM_xmlhttpRequest` 进行跨域数据采集。
- 所有网络请求均通过并发任务队列管理，保证流畅体验。
